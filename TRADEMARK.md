## Overview

This document outlines the terms for using AriaType trademarks and brand assets. While AriaType's source code is open source under GNU Affero General Public License v3.0 (AGPL-3.0), AriaType's trademarks, logos, and brand identity are separately protected.

## What are Trademarks

AriaType trademarks include but are not limited to:
- "AriaType" name
- AriaType logo and icons
- AriaType product names and taglines
- AriaType visual design elements and brand identity

## Permitted Uses

### Uses Without Permission

You may use AriaType trademarks without explicit permission in these cases:

1. **Accurate Description**: Using trademarks when describing or referencing AriaType software
   - Example: "This application is built on AriaType"
   - Example: "Compatible with AriaType"

2. **Educational and News**: Using in articles, tutorials, reviews, or news coverage
   - Must accurately and objectively describe AriaType
   - Must not imply official endorsement or affiliation

3. **Community Discussion**: Mentioning AriaType in forums, social media, or technical discussions
   - Must clearly state you don't represent AriaType officially

### Uses Requiring Permission

You must obtain written permission for:

1. **Commercial Product Naming**: Using AriaType trademark in product or service names
2. **Domain Registration**: Registering domains containing AriaType
3. **App Store Publishing**: Publishing apps using AriaType name or logo
4. **Merchandise Sales**: Using AriaType trademark on T-shirts, stickers, etc.

## Prohibited Uses

The following uses are explicitly prohibited:

1. **Confusing Use**: Using trademarks in ways that may confuse users
   - Don't imply your project is an official AriaType project
   - Don't imply your project has official AriaType endorsement

2. **Disparaging Use**: Using trademarks in disparaging, defamatory, or negative ways

3. **Modifying Trademarks**: Modifying, distorting, or creating derivative versions of AriaType trademarks

4. **Primary Identifier**: Using AriaType trademark as your product's primary identifier
   - Fork projects must use different names and logos
   - Example: If you fork AriaType, name it "YourName Voice Transcriber" not "AriaType Fork"

## Fork and Derivative Work Guidelines

If you create a fork or derivative work based on AriaType source code:

### Must Do

1. **Use Different Name**: Choose a unique name for your project
2. **Create Own Brand**: Design your own logo and visual identity
3. **Clarify Relationship**: You may state "Based on AriaType" or "Fork of AriaType"
4. **Retain Copyright Notices**: Keep original copyright and license notices in source code
5. **Follow AGPL-3.0**: Ensure your derivative work also complies with AGPL-3.0

### Must Not Do

1. **Don't Use AriaType Name**: Don't name your project "AriaType [Something]"
2. **Don't Use AriaType Logo**: Don't use AriaType's logo or icons in your project
3. **Don't Imply Official Affiliation**: Don't imply your project is official or officially supported

### Recommended Attribution

In your README or About page, you can state:

```markdown
This project is based on [AriaType](https://github.com/joe223/aria-type),
an open-source voice-to-text application. This project is an independent fork
and is not affiliated with the official AriaType team.
```

## Trademark Usage Examples

### ✅ Correct Usage

- "Our app uses AriaType engine for speech recognition"
- "Tutorial: How to Install AriaType"
- "Comparison of AriaType Alternatives"
- "Custom voice-to-text solution based on AriaType"

### ❌ Incorrect Usage

- "AriaType Pro" (implies official product)
- "AriaType for Business" (implies official version)
- "Buy AriaType License" (misleading commercial use)
- Using AriaType logo as your product icon

## Why Protect Trademarks

We protect AriaType trademarks to:

1. **Protect Users**: Ensure users can identify genuine AriaType software
2. **Maintain Quality**: Prevent low-quality or malicious software from impersonating AriaType
3. **Build Trust**: Maintain AriaType brand integrity and reputation
4. **Support Open Source**: Support project sustainability through brand differentiation
