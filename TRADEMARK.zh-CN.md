# AriaType 商标使用政策

## 概述

本文档规定了 AriaType 商标和品牌资产的使用条款。虽然 AriaType 的源代码在 GNU Affero General Public License v3.0 (AGPL-3.0) 下开源，但 AriaType 的商标、徽标和品牌标识受到单独保护。

## 什么是商标

AriaType 商标包括但不限于：
- "AriaType" 名称
- AriaType 徽标和图标
- AriaType 产品名称和标语
- AriaType 的视觉设计元素和品牌标识

## 允许的使用

### 无需许可的使用

以下情况下，您可以使用 AriaType 商标而无需获得明确许可：

1. **准确描述**：在描述或引用 AriaType 软件时使用商标
   - 示例："本应用基于 AriaType 构建"
   - 示例："与 AriaType 兼容"

2. **教育和新闻用途**：在文章、教程、评论或新闻报道中使用
   - 必须准确、客观地描述 AriaType
   - 不得暗示官方背书或关联

3. **社区讨论**：在论坛、社交媒体或技术讨论中提及 AriaType
   - 必须明确说明您不代表 AriaType 官方

### 需要许可的使用

以下情况下，您必须获得书面许可：

1. **商业产品命名**：在产品或服务名称中使用 AriaType 商标
2. **域名注册**：注册包含 AriaType 的域名
3. **应用商店发布**：使用 AriaType 名称或徽标发布应用
4. **商品销售**：在 T 恤、贴纸等商品上使用 AriaType 商标

## 禁止的使用

以下使用方式明确禁止：

1. **混淆性使用**：以可能导致用户混淆的方式使用商标
   - 不得暗示您的项目是 AriaType 官方项目
   - 不得暗示您的项目获得 AriaType 官方背书

2. **贬损性使用**：以贬损、诽谤或负面方式使用商标

3. **修改商标**：修改、变形或创建 AriaType 商标的衍生版本

4. **主要标识**：将 AriaType 商标作为您产品的主要标识
   - Fork 项目必须使用不同的名称和徽标
   - 示例：如果您 fork AriaType，应命名为"YourName Voice Transcriber"而非"AriaType Fork"

## Fork 和衍生作品指南

如果您基于 AriaType 源代码创建 fork 或衍生作品：

### 必须做的事情

1. **使用不同的名称**：为您的项目选择一个独特的名称
2. **创建自己的品牌**：设计自己的徽标和视觉标识
3. **明确说明关系**：可以说明"基于 AriaType"或"AriaType 的 fork"
4. **保留版权声明**：保留源代码中的原始版权和许可证声明
5. **遵守 AGPL-3.0**：确保您的衍生作品也遵守 AGPL-3.0 许可证

### 不能做的事情

1. **不要使用 AriaType 名称**：不要将您的项目命名为"AriaType [Something]"
2. **不要使用 AriaType 徽标**：不要在您的项目中使用 AriaType 的徽标或图标
3. **不要暗示官方关联**：不要暗示您的项目是 AriaType 官方项目或获得官方支持

### 推荐的归属方式

在您的 README 或关于页面中，您可以这样说明：

```markdown
本项目基于 [AriaType](https://github.com/joe223/AriaType) 开发，
AriaType 是一个开源的语音转文字应用。本项目是独立的 fork，
不隶属于 AriaType 官方团队。
```

## 商标保护的原因

我们保护 AriaType 商标是为了：

1. **保护用户**：确保用户能够识别真正的 AriaType 软件
2. **维护质量**：防止低质量或恶意软件冒充 AriaType
3. **建立信任**：保持 AriaType 品牌的完整性和声誉
4. **支持开源**：通过品牌差异化支持项目的可持续发展
